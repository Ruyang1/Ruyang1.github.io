乐尚代驾微服务项目


阅读数: 次    阅读数：次 2024-12-25
字数统计: 15.8k字   |   阅读时长≈ 71分

司机端：进行身份认证(身份证和驾驶证)、人脸识别模型的操作是通过腾讯云的云服务功能实现。

一、项目介绍

1.背景和功能

爱尚代驾是一种新型的出行服务模式，主营业务：商务代驾，长途代驾。其主要的特点是通过线上平台为用户提供代驾服务，伴随着中国家庭汽车保有量的飞速增长，互联网代驾行业驶进了快车道，当前项目就是基于此背景设计出来的。

2.技术概述：

后端技术栈：

  • SpringBoot：简化Spring应用的初始搭建以及开发过程
  • SpringCloud：基于Spring Boot实现的云原生应用开发工具，SpringCloud使用的技术（Spring Cloud Gateway、Spring Cloud Task、Spring Cloud Feign）
  • Spring Boot+SpringCloudAlibaba（Nacos、Sentinel）+OpenFeign+Gateway
  • MyBatis-Plus：持久层框架，也依赖于mybatis
  • Redis：内存做缓存
  • Redission：基于redis的Java驻内存数据网络-框架，操作redis的矿建
  • MongoDB：分布式文件存储数据（例如：实时记录司机的位置信息）
  • RabbitMQ：消息中间件；大型分布式项目是标配；分布式事务最终一致性
  • Seata：分布式数据
  • Drools：规则引擎（例如：起步价3公里10元，每多一公里加2元。司机等待时间超过5分钟，额外加5元。）。计算预估费用、取消费用等
  • GEO：GPS分区定位计算（例如：计算司机距离客户的路程）
  • ThreadPoolExecutor+CompletableFuture：异步编排，线程池来实现异步操作，提高效率

前端技术栈：

  • UniApp
  • Vue3全家桶
  • TypeScript
  • GraceUI
  • UniUl
  • uniapp-axios-adapter

云服务：

序号 云服务名称 具体用途
1 对象存储服务（COS） 存储司机实名认证的身份证和驾驶证照片等隐私图片
2 人脸识别（AiFace） 每天司机接单前的身份核实，并且具备静态活体检功能
3 人员库管理（face-lib） 云端存储司机注册的时候的人脸模型，用于身份比对使用
4 数据万象（ci） 用于监控大家录音文本内容，判断是否包含暴力和色情行为
5 OCR证件识别 用于OCR识别和扫描身份证、驾驶证信息
6 微信同声传译插件 把文字合成语音，播报订单；把录音转换成文本，用于安全监控
7 路线规划插件 用于规划小程序订单显示的路线、司机下班回家的路线。
8 地图选点插件 用于小程序上面的地图选点操作
9 腾讯位置服务 路线规划、定位导航、历程和时间预估

4.业务流程

乘客端：登录—-选择代驾地址(出发点和终点)—-预览地址距离、费用、时间—-呼叫代驾(生成订单、搜索附近5公里可以接单的司机)—-等待司机接单(15分钟没有司机接单，则取消订单；反之继续下面的操作)—-司乘同显—-订单开始、结束，乘客根据账单进行支付(可能有优惠券)

司机端：登录—-进行身份认证(身份证、驾驶证)（通过云服务实现）—-开始接单—-抢单—-前往代驾地点、开始代驾—-代驾结束—-发送账单给乘客

管理员端：创建司机人脸识别模型，生成支付账单(例如：高速费、停车费、规则引擎)，发送账单信息给司机

3.技术点：Mybatis-Plus

3.1Mybatis_Plus概述

  • 官方网站：baomidou.com
  • MyBatis-Plus 是一个 MyBatis 的增强工具，在 MyBatis 的基础上只做增强不做改变，为简化开发、提高效率而生。
  • 使用Mybatis-Plus，如果实现单表操作，不需要编写sql语句，封装方法，使用封装的这些方法方便实现；如果是复杂查询或者多表查询的话，还得自己编写sql语句。

Mybatis-Plus特性：

  • 无侵入：只做增强不做改变，引入它不会对现有工程产生影响，如丝般顺滑
  • 损耗小：启动即会自动注入基本 CURD，性能基本无损耗，直接面向对象操作
  • 强大的 CRUD 操作：内置通用 Mapper、通用 Service，仅仅通过少量配置即可实现单表大部分 CRUD 操作，更有强大的条件构造器，满足各类使用需求
  • 支持 Lambda 形式调用：通过 Lambda 表达式，方便的编写各类查询条件，无需再担心字段写错
  • 支持主键自动生成：支持多达 4 种主键策略（内含分布式唯一 ID 生成器 - Sequence），可自由配置，完美解决主键问题
  • 支持 ActiveRecord 模式：支持 ActiveRecord 形式调用，实体类只需继承 Model 类即可进行强大的 CRUD 操作
  • 支持自定义全局通用操作：支持全局通用方法注入（ Write once, use anywhere ）
  • 内置代码生成器：采用代码或者 Maven 插件可快速生成 Mapper 、 Model 、 Service 、 Controller 层代码，支持模板引擎，更有超多自定义配置等您来使用
  • 内置分页插件：基于 MyBatis 物理分页，开发者无需关心具体操作，配置好插件之后，写分页等同于普通 List 查询
  • 分页插件支持多种数据库：支持 MySQL、MariaDB、Oracle、DB2、H2、HSQL、SQLite、Postgre、SQLServer 等多种数据库
  • 内置性能分析插件：可输出 SQL 语句以及其执行时间，建议开发测试时启用该功能，能快速揪出慢查询
  • 内置全局拦截插件：提供全表 delete 、 update 操作智能分析阻断，也可自定义拦截规则，预防误操作

三、乘客端登录

乘客端只需要登录，不需要注册。因为是微信小程序方式登录，
我们根据用户登录时候提交的接口，获取到乘客微信的OpenId(唯一的)，然后到数据库中查询该OpenId是否存在，如果已经存在，则根据用户信息生成token返回，直接登陆即可；如果不存在，则自动为乘客注册，并将该记录添加到乘客表中。

3.1微信小程序登录流程

  1. 微信小程序发起登陆请求，腾讯接口返回code（临时票据）
  2. 后端接口获取code值
  3. 后端接口返回三个值（小程序id+密钥+code）请求腾讯接口
  4. 腾讯接口返回包含openid的信息（openid是微信账号的唯一标识）
  5. 通过openid判断用户是否需要注册（如果是第一次登陆自动帮其注册；如果不是第一次登陆，生成token值并返回）

3.4登录校验

  1. 分析过程

    • 如何判断是否登陆状态？

      —判断请求头里面是否包含token字符串

      —根据token查询redis

    • 如何实现？

      —自定义注解+aop进行优化

  2. 自定义注解：**@GuiguLogin**：哪个controller需要登录校验，就在这个controller的方法上，添加自定义注解

  3. aop：判断哪个方法上面有**@GuiguLogin**这个自定义注解，当执行该方法时，进行登录校验操作（判断token和redis是否有数据）

    补充：aop：面向切面编程：在不改变源代码或者很少改变的情况下，增强类中的某些方法。
    aop操作术语：如下

    切入点 实际被增强的那些方法
    增强（通知） 实际增加的功能，比如待会需要增加的登陆判断（有5钟类型，其中环绕通知最常用）
    切面 把增强使用到切入点的过程就是切面（我理解就是实际敲写的代码）

    ​ aop具体操作：如下

    • 第一步：创建切面类

    • 第二步：在切面类中创建方法，在方法上添加注解，实现不同类型的通知（建议使用环绕通知实现）
      使用切入点表达式，对那些方法进行增强。

    • 第三步：在环绕通知方法里面，写具体逻辑

      *在具体业务方法执行之前，进行登陆判断

      *判断逻辑：判断请求头是否有token，根据token查询redis

5.4.2费用规则

代驾费用=里程费+等候费+远途费

  1. 里程费=基础里程费+超出起步里程的费用

    时间段 基础里程 收费 超出起步里程的费用
    00:00~16:00 3公里 19元 4公里/1元
    06:00~22:00 5公里 19元 3公里/1元

  2. 等候费：免费等候10分钟，每超过1分钟加收1元

    10分钟之内 10分钟之后
    免费 1元/1分钟

  3. 远途费：行程小于12公里不加收，超过12公里视为远途，每超过1公里加收1元

    12公里以内 12公里以外
    不加收费用 1元/公里

注意：规则随时会调整，因为实际情况不同。例如：油价上涨，雨雪天气，高速费、停车费等。

七、任务调度（定时任务）

7.1需求

引出问题：乘客下单之后，会搜索附近可以接单的司机。如果附近没有符合条件的司机，怎么办？
答案：乘客下单之后，会不断进行搜索，每1分钟搜索一次，这需要项目使用任务调度，但是我们不可能一直搜索，所以规定草果15分钟之后，就取消该订单。
方法：通过任务调度搜索到司机之后，需要在服务器端给这些司机建立临时队列，把新订单数据放到每个司机的队列中。司机开启接单服务之后，能在队列里面看到订单数据，然后选择是否抢这单。
